## pi-agent-control — Overview

A workflow system for automating, recording, composing, and verifying demos/QA proofs of the Pi Coding Agent CLI — and other terminals/browsers.

### Routing (3 lookups)

| Route | Question | Skills it loads |
|---|---|---|
| **Target** | What are you driving? | `pi-agent-cli`, `tuistory`, `true-input`, `agent-browser`, `pty-capture` |
| **Stage** | What does the workflow need? | `capture`, `compose`, `verify` |
| **Artifact** | Polish needed? | `showcase` (window chrome, branding) |

### Workflow shape

```
Command (intent + commitments)
  → Target route (load driver atoms)
  → Capture (record / screenshot / byte-capture)
  → Compose (assemble deliverable, if needed)
  → Verify (check against commitments)
  → Report
```

### Key rules

1. **Use `tctl` (not raw `tuistory`)** for any recorded session — `tuistory` has no recording capability
2. **Isolate every run** — generate a `RUN_ID` and `RUN_DIR`, prefix all session names
3. **Use workers for capture and render tasks** — parent plans, worker executes exact commands
4. **Must set `FORCE_COLOR=3` + `COLORTERM=truecolor`** for colored terminal captures
5. **Layout defaults**: `single` for new features/walkthroughs, `side-by-side` for regressions/comparisons

### Layout ↔ viewport sizing

| Layout | Panel aspect | Recommended browser viewport |
|---|---|---|
| `single` | ~16:9 landscape | **1280×720** or **1440×810** |
| `side-by-side` | ~8:9 near-square | **960×1000**, **900×1000**, **1024×1080** |
